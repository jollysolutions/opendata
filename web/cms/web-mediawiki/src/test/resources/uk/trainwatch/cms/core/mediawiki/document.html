Welcome to piweather.center

This is the project site for my piweather.center project to build a Raspberry PI Weather Station from scratch.

Background

This project started shortly after the Raspberry PI was released. The Mark 1 station was based around PYWWS and ran for 3 years until mid July 2015 when it suffered terminal damage from a Bird Strike smashing the external sensors.

The Mark 2 station was originally aimed for release during 2014. I had written the software and had a prototype cloud cover sensor up and running. However other projects and the day job (also the commute) affected this so I've not had time to resume work on it.

At the time of writing this, it's the end of November 2015 and I'm starting to resume work on it. The release of the new Raspberry PI Zero this week was also the final nudge in resuming work on this project.

Which models will be supported

This will go on it's own page but the following matrix will show what model's of Raspberry PI will be supported (or at least planned):

Model A, A+, B rev2, B+, 2B Zero
Wind Speed Yes Yes
Wind Direction Planned Planned
Rain Fall Yes Yes
Pressure (I2C) Yes Yes
Light Level (I2C) Yes Yes
Arduino Slaves (I2C) Yes Yes
Temperature Planned Planned
Cloud Coverage Yes No/Maybe

Notes

  • Once I've figured out the best way to get the Anenometer working I will then look at the Wind Direction which is trickier to implement.
  • For temperature, I've used different censors but using a third party board which is no longer available, so for the project restart I'm going to use a combination of different sensors. Some use I2C so will be simple to add but I also want to use 1wire sensors. The latter will allow not just air temperature but ground, road & rail temperatures to be measured as well.
  • Cloud cover will not be supported on the Zero simply because it doesn't have a CSI port, so I can't use the standard camera like the prototype does. However a USB or TTL/Serial camera are two options.
